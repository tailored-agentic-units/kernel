## Key Findings

### Decomposition Architecture

Three independent libraries with kernel as composition layer:

```
tau/agent (primitives — zero TAU deps)
tau/orchestrate (coordination — zero TAU deps)
    ↓ both consumed by ↓
tau/kernel (harness — depends on both)
```

**Critical discovery**: `orchestrate/hub` only uses `ag.ID()` from `agent.Agent`. A local `Participant` interface decouples tau/orchestrate entirely from tau/agent.

### go-agents v0.5.0 Divergence

Major improvements need incorporation into tau/agent:
- Format abstraction layer (wire format registry: OpenAI, Converse)
- Streaming transport abstraction (SSE, EventStream)
- AWS Bedrock provider with SigV4 auth
- Unified Response type with ContentBlock interface
- Identity/credential management package

### Package Disposition

| Destination | Packages |
|-------------|----------|
| tau/agent | core/config, core/protocol, core/response, core/model, agent/*, + new format/, streaming/, identities/ |
| tau/orchestrate | observability/, orchestrate/* |
| Kernel retains | kernel/, session/, memory/, tools/, mcp/, cmd/ |
| Remove | rpc/ (dead ConnectRPC infrastructure) |

## Actions Taken

- Comprehensive codebase audit across kernel, go-agents, go-agents-orchestration, tau-marketplace

- Documented all findings in plan file (`.claude/plans/clever-bubbling-naur.md`)

## Decisions Made

1. **Phase 1A (Library Extraction) before resuming #26-28** — avoid throwaway work
2. **Flat package layout** for tau/agent — root package defines Agent interface
3. **Best-of-both-worlds protocol** — adopt go-agents v0.5.0 format/streaming/response while preserving kernel innovations (registry, multi-turn)
4. **Orchestrate independence** — local Participant interface instead of agent import
5. **All standalone marketplace plugins** — no more monolithic tau plugin
6. **Kernel changes deferred** — no project management or doc changes until new infrastructure proves the architecture

## Recommendations

1. **Next session**: Concept development (`/dev-workflow concept`) to formalize library extraction architecture
2. **Preserve kernel's agent registry** in tau/agent — absent from go-agents v0.5.0
3. **Deep protocol reconciliation** during concept session — central design challenge
4. **Team onboarding** after Phase A infrastructure, before Phase B refactoring
